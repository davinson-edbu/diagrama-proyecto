 tipo_documento : Entity 		Tabla maestra que parametriza los diferentes tipos de documentos de identidad permitidos en el sistema (por ejemplo: Cédula de Ciudadanía, Cédula de Extranjería, Pasaporte, etc.).
 servicio : Entity 		Catálogo de servicios que el taller ofrece a sus clientes.
 producto : Entity 		Almacena los productos, repuestos o materiales utilizados durante los trabajos realizados en el taller.
 orden_trabajo : Entity 		Representa una orden de trabajo generada para un cliente, donde se registran los servicios y productos utilizados durante la atención de un vehículo.
 moto : Entity 		Almacena la información principal de las motocicletas registradas en el sistema..
 cliente : Entity 		Almacena la información de control específica para los usuarios con el rol de cliente, registrando su ubicación, estado en la plataforma y fidelización por puntos.
 cuenta : Entity 		Almacena la información de perfil, datos personales e identificación del cliente o usuario vinculado a una cuenta del sistema.
 factura : Entity 		Representa el documento de cobro generado a partir de una orden de trabajo finalizada. Contiene el detalle de los valores económicos asociados a los servicios y productos utilizados, incluyendo impuestos y total a pagar.
 user_authority : Entity 		Tabla intermedia (asociativa) utilizada para romper la relación de Muchos a Muchos entre las entidades user y rol. Permite asignar uno o múltiples roles a un mismo usuario.
 user : Entity 		Almacena las credenciales de autenticación, llaves de activación/recuperación y el estado de acceso de cada usuario en la plataforma.
 rol : Entity 		Almacena los diferentes roles o perfiles del sistema (por ejemplo: Administrador, Almacenista, Cliente, Mecánico) para gestionar los permisos de acceso de los usuarios.
